# Отчёт по лабораторной работе №2

**Дисциплина:** Проектирование программного обеспечения интеллектуальных систем (часть 2)  
**Вариант:** 5  
**Тема:** Учёт пациентов поликлиники  

## 1. Постановка задачи

Разработать оконное приложение на Python с GUI для ведения массива записей о пациентах. Каждая запись содержит:

| Поле | Тип |
|------|-----|
| ФИО пациента | строка |
| Адрес прописки | строка |
| Дата рождения | `date` |
| Дата приёма | `date` |
| ФИО врача | строка |
| Заключение | строка |

Условия поиска и удаления (вариант 5):

1. По фамилии пациента **или** адресу прописки.
2. По дате рождения.
3. По элементам ФИО врача (достаточно одного, например только имя) **или** по дате приёма.

## 2. Архитектура MVC

| Слой | Каталог | Назначение |
|------|---------|------------|
| **Model** | `src/model/` | `PatientRecord`, `PatientRepository`, SQLite, XML (DOM/SAX), логика сопоставления `criteria.py` |
| **View** | `src/view/` | Главное окно, диалоги, виджеты пагинации и выбора даты |
| **Controller** | `src/controller/` | `AppController` — обработка команд меню и обновление представления |

Пользовательские действия идут: View → Controller → Model → Controller → View.

## 3. Реализованный функционал

- Главное окно с таблицей записей и постраничным выводом (по умолчанию 10 записей).
- Панель пагинации: первая/предыдущая/следующая/последняя страница, изменение размера страницы, отображение номера страницы, общего числа страниц и записей.
- Альтернативный режим «дерево»: запись — узел, поля — дочерние листья.
- Меню и дублирующая панель инструментов.
- Диалог добавления записи с календарём для дат (`QDateEdit` с `calendarPopup`).
- Отдельный диалог поиска с вводом условий и таблицей результатов.
- Отдельный диалог удаления с сообщением о количестве удалённых записей или об отсутствии совпадений.
- Хранение в SQLite (`data/patients.db`).
- Загрузка/сохранение XML: запись через `xml.dom.minidom`, чтение через `xml.sax`.
- Экспорт/импорт в файл SQLite по стандартному диалогу выбора файла.

## 6. Сценарии демонстрации

### 6.1. Загрузка демонстрационных данных

1. Запустить приложение.
2. Меню **Файл → Загрузить XML…** → выбрать `data/demo/patients_clinic_1.xml`.
3. Убедиться, что в главном окне отображаются записи, в статусе пагинации — «всего записей: 55».

### 6.2. Просмотр и пагинация

1. Перейти на страницы 2, 3 кнопками «>» и «>|».
2. Изменить «Записей на странице» на 5 — таблица обновится, число страниц увеличится.

### 6.3. Режим дерева

1. Меню **Вид → Отображение деревом** (или кнопка «Дерево» на панели).
2. Развернуть узел записи — видны все поля как дочерние элементы.

### 6.4. Добавление записи

1. **Записи → Добавить**.
2. Заполнить осмысленные данные, выбрать даты в календаре, сохранить.
3. Новая запись появляется в таблице.

### 6.5. Поиск

1. **Записи → Поиск**.
2. Режим «По фамилии…» — ввести `Иванов`, нажать **Найти** — в таблице диалога только подходящие записи.
3. Режим «По дате рождения» — выбрать дату из существующей записи.
4. Режим «По ФИО врача…» — ввести только имя `Елена`, найти записи врача Смирновой.

### 6.6. Удаление

1. **Записи → Удалить**.
2. Задать условие (например, адрес содержит `Сурганова`), подтвердить.
3. Получить сообщение «Удалено записей: N» или «не найдены».

### 6.7. Сохранение в XML и БД

1. **Файл → Сохранить XML…** — сохранить копию массива.
2. Загрузить другой файл `patients_clinic_2.xml` — данные заменятся.
3. **Файл → Сохранить БД…** — экспорт в `.db`, затем **Загрузить БД…** для восстановления.

## 7. Используемые технологии
